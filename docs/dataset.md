# MilCOD7K Dataset

A large-scale benchmark for **multi-class military camouflage object detection/segmentation**.

- **7,200** images, **pixel-level** annotations
- **4 foreground classes** + background: camouflage soldier, military vehicle, tank, fortification
- **Real + AI-generated mix:** 4,944 real photos + 2,256 AI-generated images
- **Splits (8:1:1):** train 5,760 · val 720 · test 720
- Terrains: woodland, desert, urban, mixed; scenes: day/night/snow, etc.

> 📄 **Full datasheet** (data sources, real/AI split rule, annotation protocol, class definitions, deduplication, license & ethics) is in [`DATASET.md`](../DATASET.md). A per-image label file `metadata.csv` (columns: `filename, split, class, class_id, source` ∈ {real, ai}) ships with the dataset archive.

| 0 | background | 88.45% | — |
| 1 | camouflage_soldier | 1.68% | 6.81% |
| 2 | military_vehicle | 3.72% | 15.17% |
| 3 | tank | 1.45% | 5.92% |
| 4 | fortification | 4.70% | 16.72% |

Foreground totals 11.55% of pixels — an inherent camouflage-scene imbalance that motivates the class-weighted loss in training.

## 3. Native format

The released dataset uses our loader's native layout:

```
MilCOD7K/
├── train/{Imgs,GT,Edge}/
├── val/{Imgs,GT,Edge}/
└── test/{Imgs,GT,Edge}/
```
- `Imgs/*.jpg` — RGB images.
- `GT/*.npy` — `uint8` array of shape **H×W×5**, a per-class one-hot mask (channel c = 1 where pixel belongs to class c).
- `Edge/*.png` — single-channel boundary maps (used by boundary-aware losses).

### Converting to other layouts

The same data can be turned into common layouts with the scripts in [`tools/`](../tools/):

- **Class-index PNG** (single-channel, pixel = class id 0–4; Cityscapes/PASCAL/MMSeg style):
  ```bash
  python tools/npy_to_classpng.py --in data/MilCOD7K --out data/MilCOD7K_std
  ```
- **YOLOseg polygons** (`class_id x1 y1 ... xn yn`, normalized) — from the source annotations:
  ```bash
  python tools/convert_yolo_to_multiclass.py \
      --yolo-img data/MilCOD7K_yolo/images --yolo-label data/MilCOD7K_yolo/labels \
      --out data/MilCOD7K
  ```

## 4. Real vs AI-generated, and interference images (`metadata.csv`)

`metadata.csv` (auto-generated by `tools/build_metadata.py`) labels every image:

```csv
filename,split,class,class_id,source,interference
cs_mcod00001.jpg,train,camouflage_soldier,1,real,False
f_mcod1196.jpg,train,fortification,4,ai,False
i_t_mcod00089.jpg,train,tank,3,real,True
...
```

The real/AI rule (matches the expected counts — train 3986/1774, val 420/300, test 538/182):

- `_mcod` followed by **exactly 5 digits** (zero-padded, e.g. `mcod00159`) → **real**
- `_mcod` followed by **fewer than 5 digits** (e.g. `mcod1196`) → **ai**

The camouflage soldier class is **all real** (0 AI images); AI generation is concentrated in fortification > vehicle > tank.

Filter the `source` column to reproduce the real/AI ablation: `real-only`, `ai-only`, or `mixed`.

### Interference images (`i_` prefix)

A subset of images have an `i_` prefix (e.g. `i_t_mcod00089.jpg`). This marks an **interference** dimension — images (both real and AI-generated) with additional interference applied as a robustness/augmentation variant. It is independent of the real/AI split: an `i_` image's real/AI label still follows the `_mcod` digit rule, and the `i_` flag is recorded separately in the `interference` column so the two dimensions can be studied independently. Counts: ~960 in train, ~120 each in val/test; within those, both real and AI sources are present (e.g. in train, ~695 real + ~265 AI).
